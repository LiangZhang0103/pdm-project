## 3. 可用性需求

### 3.1 系统可用性

| 指标 | 目标 |
|------|------|
| 运行时间 | 99.5% |
| 计划维护 | 月度，最大4小时 |
| MTBF | > 720小时 |
| MTTR | < 1小时 |

### 3.2 容错能力

| 场景 | 预期行为 |
|------|----------|
| 数据库连接失败 | 返回503，快速失败 |
| MinIO不可用 | 缓存降级，显示警告 |
| API超时 | 重试3次，失败返回错误 |
| 前端网络断开 | 显示离线状态 |

## 4. 安全性需求

### 4.1 认证

| 要求 | 实现 |
|------|------|
| 密码最小长度 | 6字符 |
| 密码哈希 | bcrypt (cost 12) |
| Token过期 | 1小时 |
| 登录失败限制 | 5次/5分钟 |

### 4.2 授权

| 要求 | 实现 |
|------|------|
| 基于角色 | user, admin |
| 最小权限 | 按需分配 |
| 权限检查 | 所有写操作 |

### 4.3 数据保护

| 数据类型 | 保护措施 |
|----------|----------|
| 密码 | bcrypt哈希 |
| Token | 不存储敏感信息 |
| API密钥 | 环境变量 |
| 用户数据 | 角色隔离 |

### 4.4 传输安全

| 环境 | 要求 |
|------|------|
| 开发 | HTTP可选 |
| 生产 | HTTPS强制 |
