## 技术要点提醒

### 数据库设计原则
1. **规范化**：避免数据冗余
2. **索引优化**：为查询频繁的字段添加索引
3. **外键约束**：保证数据完整性
4. **迁移策略**：使用Alembic进行数据库迁移

### 安全注意事项
1. **密码安全**：使用bcrypt哈希，加盐
2. **API保护**：JWT token过期时间设置
3. **文件上传**：限制文件类型和大小
4. **SQL注入**：使用ORM或参数化查询

### 性能优化
1. **数据库连接池**：合理配置连接数
2. **缓存策略**：Redis缓存热点数据
3. **分页查询**：大数据量时分页
4. **前端懒加载**：按需加载资源
